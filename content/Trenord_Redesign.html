Trenord Online Portal Redesign

Usability oriented redesign - January 2020

The main goal of the project was the web site redesign of the train company Trenord. The ultimate goal of this work was the improvement of the user usability of the web portal. The whole process was guided by user center design methodologies, where usersâ€™ needs and frustrations are the main focus of the project, these methods were then helped by heuristic and quantitative evaluations followed by benchmarking with competitors to further advance the redesign process.

Personas and Users tasks

Being the process guided by the user centered approach, the first phase of the project was dedicated to understanding the users of the service, so five people have been defined through field research. Then by analyzing their needs and frustrations, a list of possible tasks that these users could use in the interaction with Trenord portal were defined. We then defined which of these tasks were the most important for our personas, so that we could analyze them better in the next project phases.

Definition of competitors and benchmarking

After defining the most important tasks for our personas, we started to analyze the website. We initially look at the state of the art of the Trenord site, analyzing it both in quantitative terms (number of pages to reach the goal of the task, time taken, number of clicks ...), and in qualitative terms (using parameters such as: number of languages, user support, error preventions ...). Subsequently, three "competitors" operating in the same sector or in adjacent sectors were defined, and the same analysis with the same parameters was made. The results allowed us to generate useful insights and understand what we had to fix.

First phase of redesign

After defining the main area of the website that needed to be fixed, we stared the main phase of redesign, beginning with the map of the website architecture for each task. Then we changed this architecture so that the information and functions offered by the site cloud respond more appropriately to the needs of users, defining the main flow of the interactions. Then we stated to sketch the main wireframes in order to understand where we could put all the main features and information inside the page, keeping an eye on the users needs and overall usability of the platform. At the same time, following the guidelines of the coordinated brand image, we defined the main style guidelines, in order to guarantee uniformity in every element of the web portal.
