UX Mapping

Studies on UX in the transport sector - December 2018

These exercises (generated within the User Experience course at the Politecnico di Milano) were aimed at mapping the various experiences that different types of services (all developed around the people’s need to move) create for end users. Here there is the part of the work made regarding BlaBlaCar service.
This brand was particular suited as case study, considering that it’s a new service that provide a new way for the users to interact with each other through the use of technology.

System map and touchpoint

The first stages of this analysis did not immediately focus on the user himself, but instead revolved around the how and where the company provided the service to the users, what backstage activities supported them and which touchpoints and channels the user could use to interact with the brand during his journey.
In doing so, the "flows" of data, financial resources and services were mapped, to understand how the service could sustain itself and provide the best possible experience for the end user.

Digital ethnography, user flow, journey and experience map

A Digital ethnography was conducted on the users’ comments on several social media (Facebook, Twitter, Google Playstore ...), both "official" (i.e. managed by the brand) and not.
This was helpful in order to understand what opinions, motivations, mental frameworks and perceived pain points the users had about the service.
From the analysis of these data we produced a foundational understanding of the users, on which different services maps were created. An "ideal" user flow was outlined as the basis for the experience map, in which the main comments of the digital ethnography were integrated in order to be able to identify the main problems that the user experience within the single ride.

Mental Model Map

Finally, we built a mental model map showing what the user could feel and think at all times of the trip, both in positive and negative terms.
From here we then conducted a gap analysis in order to understand if the system was providing all the “tool” to support the users on their journey, identifying varius opportunities for improve the system.
This final map in association with the previous ones made up a fairly complete framework about the users’ UX offered by the BlaBlaCar service at that time.
