Interfaces for building trust in Autonomous Vehicles

Interaction methods for autonomous vehicles - September 2019

In the next 10 years, new type of services for people transportation are expected to arise, these services will be made possible thanks to the development of self-driving vehicles. Unfortunately, these types of services will face a huge problem: the lack of users’ trust about this type of technologies. The main aim of the project was to create interfaces to support the formation of trust between the user and the vehicle. Thus, the main methods of interaction between user and vehicle were defined based on research already conducted on the matter. A VR prototype was subsequently produced to test these new interactions with the users, and to understand which of these methods are the most effective.

First research, problem setting and possible solutions

The first step of the project was to conduct a desk research on how to build trust in self-driving cars. Based on the knowledge gathered, we then proceed to define the main features that the machine interface should have, one of which was communicating the right amount of information to the user in the right way. Once we identified all the possible data the user needed, we then defined how this information should be communicated to the passenger and what interactions it could have with the car.

After defining all the possible channels of human/machine interaction, we concentrated on their actual development. Four main interfaces have been defined: a large central touch screen divided into 3 parts (a central part for display the main navigation data, and two external ones for the passengers use), a top bar to display the AI messages, complimented with a voice interface with which passengers could communicate with the car, and finally a "windshield screen" for the advertising and entertainment part.

Since the machine is a touchpoint of a much bigger service, a user journey with the vehicle and the service has also been sketched, from when the user waits for the arrival of the vehicle to when the ride ends.

Prototype creation

We then built a working prototype using Unity3D, HTC Vibe VR, Microsoft Voice Recognition and Motion Leap with the aim of testing our solutions with real users. The choice to use these tools was forced by the nature of the project, which investigating technologies not yet ready or available on the market, required the use of virtual reality to test them.
